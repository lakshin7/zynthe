# HuggingFace Token Setup

## Quick Setup

1. **Get your token** from HuggingFace:
   - Visit: https://huggingface.co/settings/tokens
   - Click "New token"
   - Select "Read" access (or "Write" if you plan to push models)
   - Copy the token (starts with `hf_...`)

2. **Add token to `.env` file**:
   ```bash
   # Open .env file
   nano .env
   
   # Or just paste your token after HF_TOKEN=
   HF_TOKEN=hf_YourTokenHere...
   ```

3. **Restart the app**:
   ```bash
   ./start-zynthe.sh
   ```

## What the Token Does

✅ **Download Private Models** - Access gated models like Llama, Mistral, etc.
✅ **Download Models Faster** - Better download rates for authenticated users
✅ **Access Your Models** - Use your own private models from HuggingFace

## Security

- ⚠️ **Never commit `.env` to git** - The `.gitignore` already excludes it
- 🔒 **Keep your token private** - Don't share it or post it online
- 🔄 **Rotate tokens regularly** - Create new tokens and delete old ones

## Without Token

If you don't add a token, the app will still work but:
- ❌ Can't download private/gated models (Llama, Mistral, etc.)
- ✅ Can download public models (BERT, RoBERTa, DistilBERT, etc.)
- ⚠️ May have slower download speeds

## Testing

To verify your token works:

```bash
# In Python
from huggingface_hub import login
login(token="hf_YourTokenHere...")

# Should see: "Login successful"
```

Or just start training - the logs will show:
```
✅ Logged in to HuggingFace with token
```

## Troubleshooting

**"Invalid token" error:**
- Check token starts with `hf_`
- Verify token from https://huggingface.co/settings/tokens
- Make sure token has "Read" permissions

**Token not being used:**
- Restart the backend server
- Check `.env` file has no spaces: `HF_TOKEN=your_token` (not `HF_TOKEN = your_token`)
- Make sure `.env` is in project root directory

**Still can't download model:**
- Some models require accepting terms first
- Visit the model page on HuggingFace
- Click "Agree and access repository"
- Then try downloading again
